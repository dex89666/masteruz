# Инструкция: Как использовать систему привилегированных лиц в MasterUz

## Быстрый старт (для админа)

### 1️⃣ Создание профиля привилегированного лица

**Шаг 1:** Админ-панель → Privileged Officials → **+ Create**

**Заполнить:**
- **Пользователь** — найти в системе (по юзернейму или Telegram ID)
- **Организация** — макс 100 символов (例: "ТОО ЖЭК №3 Алмалинского района")
- **Должность** — макс 50 символов (例: "Начальник")  
- **Тип привилегии:**
  - ✅ **BUILDING_HEAD** — начальник ЖЭКа, УК (рекомендуется)
  - 🏛️ **OFFICIAL** — чиновник, госслужащий
  - 🏢 **BUSINESS_OWNER** — директор предприятия
  - 📍 **DISTRICT_HEAD** — начальник округа/района
- **Документ** — скан приказа / справки о назначении (опционально)
- **Срок действия** — по умолчанию 1 год

**Результат:** Профиль создаётся со статусом **PENDING** (ожидание проверки)

---

### 2️⃣ Проверка и одобрение

**Шаг 1:** Админ-панель → Privileged Officials → Найти в списке

**Шаг 2:** Нажать на профиль

**Шаг 3:** Кнопка **Approve** (если документы в порядке)

```json
{
  "status": "ACTIVE",
  "statusReason": "Документы верифицированы, приказ №1234"
}
```

**Результат:**
- ✅ Статус → ACTIVE
- 📧 Пользователю отправляется письмо с подтверждением
- 🎁 Активируются все привилегии

---

### 3️⃣ Использование привилегий (для пользователя)

Если человеку назначена привилегия, он видит в приложении:

#### 🚀 Ускорить заказ
```
Order → Menu → ⚡ Fast-Track (5 раз в день)

Причина: "Срочный ремонт в жилом комплексе, протечка"
```
Заказ переместится вверх в очереди мастеров.

#### 🎯 Назначить приоритет
```
Order → SetPriority → Уровень 1|2|3 

Custom SLA: 2 часа вместо 24 (если allowed)
Бонус за ускорение: +50,000 сум при выполнении
```

---

### 4️⃣ Мониторинг KPI

**Админ-панель → Privileged Officials → Officer Profile**

**Видеть:**
- 📊 SLA Score — % заказов выполнено в срок
- ⭐ Avg Rating — средняя оценка клиентов (1-5)
- 💰 Bonus Earned — начислено бонусов за месяц
- 📞 Total Orders — всего обработано заказов

**Пример:**
```
Сентябрь 2024:
- SLA Score: 92% (порог 85% ✅)
- Avg Rating: 4.7/5 (порог 4.0 ✅)  
- Бонус: 87,500 сум
```

---

### 5️⃣ Ежемесячный расчёт KPI

**В конце месяца (вручную):**

1. Админ-панель → Privileged Officials → Officer Profile
2. Кнопка **Calculate KPI** → Period: 2024-09
3. Система рассчитает автоматически:
   - Сколько заказов выполнено
   - Сколько в срок (SLA Score)
   - Средний рейтинг  
   - Сумму бонуса

**Результат:** Бонус переводится на баланс пользователя

## Тревоги и уведомления

### ⚠️ Когда админ должен вмешаться

1. **Низкий SLA** — если Score < 80%  
   → Пересмотреть мотивацию или помочь ускорить процесс

2. **Низкий рейтинг** — если Avg Rating < 3.5  
   → Связаться, узнать проблемы

3. **Скоро истекает** — за 30 дней  
   → Отправить напоминание о продлении

4. **Подозрение на абюз** — много fast-track без результата  
   → Проверить actions history и принять меры

## История действий (Action Log)

**Админ-панель → Privileged Officials → :officialId → Actions**

Видны все действия привилегированного лица:
- Ускорение заказов
- Назначение приоритетов
- Комментарии к заказам
- IP адреса и User-Agent

**Используется для:**
- 🔍 Аудита и контроля
- 🛡️ Выявления мошенничества
- 📋 Соответствия регламентам

## Отрыв привилегии

### Приостановка (временная)
```
Status: SUSPENDED
Reason: "Превышены лимиты ускорений"

До следующего месяца
```

### Отзыв (постоянный)
```
Status: REVOKED  
Reason: "Нарушение регламента / документы невалидны"

Постоянно
```

**Результат:** Все привилегии теряются, заказы распределяются нормально

## Часто задаваемые вопросы

**Q: Как долго действует привилегия?**  
A: По умолчанию 1 год. Можно установить любую дату в будущем при создании.

**Q: Кто видит KPI?**  
A: Только админ. Привилегированное лицо может видеть свой баланс и заказы, но не полные метрики.

**Q: Что если бонус не начислился?**  
A: Проверить:
- Был ли рассчитан KPI вручную?
- Выполнены ли все условия (SLA + Rating)?
- Есть ли заказы за период?

**Q: Можно ли изменить параметры мотивации?**  
A: Да! Admin → Motivation Settings → :privilegeType

**Q: Что если пользователь отклонил ускорение?**  
A: Лимит не расходуется, действие логируется как FAILED
